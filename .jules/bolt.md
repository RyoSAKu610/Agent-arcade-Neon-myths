## 2024-06-25 - Replace O(N) Array.find() with O(1) 2D Grid
**Learning:** When performing spatial coordinate lookups for small datasets in V8, a pre-computed 2D array grid (`arr[y][x]`) drastically outperforms both `Array.prototype.find()` (O(N) lookup) and standard Map lookups using string key interpolation (e.g., ``Map.get(`${x},${y}`)``). The overhead of creating a string template and performing hashing for the Map key is significantly higher than directly indexing a 2D array, especially when the map boundaries (`MAP_W`, `MAP_H`) are small and fixed (e.g., 22x18). This optimization is crucial for functions called repeatedly in render loops like `getBld(x, y)`.
**Action:** Always prefer pre-computing fixed 2D arrays for small, bounded grid spatial lookups instead of doing dynamic scans or string-interpolation Map lookups in highly frequently executed paths.
