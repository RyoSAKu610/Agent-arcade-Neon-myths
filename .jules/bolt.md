## 2024-05-11 - Scope ReferenceError in nested callback optimization
**Learning:** When optimizing state setters that are conditionally called inside a loop/effect block, DO NOT move inner block-scoped variables outside to the outer callback's scope. Doing so will cause a `ReferenceError` (e.g. `newConns is not defined`) when the condition is false or if lexical scoping is broken.
**Action:** While keeping `.reduce()` for mapping and filtering in a single pass is the right approach to performance, it must be applied correctly to the existing state update structure. Keep the updates separate or ensure variables are correctly scoped. Here, we kept `setConnections(prev => [...prev, ...newConns])` in the conditional block, and simplified the external cleanup using `prev.reduce` rather than merging both into one scoped closure.
